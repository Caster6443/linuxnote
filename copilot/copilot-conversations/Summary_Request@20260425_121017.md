**ai**: 这段配置实现了 **Hyprland 环境下核显（A 卡）常驻桌面，独显（N 卡）按需直通给虚拟机** 的热切换方案，并通过 libvirt 钩子完成了 **开机自动解绑、关机自动绑回** 的全自动化流程。

### 核心步骤总结

- **1. 隔离桌面与 N 卡**  
  通过 `uwsm` 启动 Hyprland，并在 `env-hyprland` 中设置 `AQ_DRM_DEVICES` 等环境变量，强制 Hyprland 自身只使用 A 卡（核显），避免桌面占用 N 卡资源。同时为 A 卡创建 udev 规则，确保设备路径稳定（`/dev/dri/amd-igpu`）。

- **2. 创建显卡切换脚本**  
  - `detach_nvidia.sh`：停止 N 卡相关服务，卸载 nvidia 驱动，并（可选）加载 vfio-pci，将 N 卡交由 VFIO 接管。  
  - `bind_nvidia.sh`：从 VFIO 剥离设备，重新加载 nvidia 驱动，让宿主机重新识别 N 卡。

- **3. 配置 libvirt 钩子**  
  在 `/etc/libvirt/hooks/qemu` 中为指定虚拟机（如 `win11`）编写脚本：
  - **开机前** (`prepare begin`)：检查 N 卡是否被其他进程占用（如未关闭的应用），若无则执行 `detach_nvidia.sh`，完成隔离并通知用户。
  - **关机后** (`release end`)：执行 `bind_nvidia.sh` 将显卡归还宿主机，并发送完成通知。

- **4. 透传 N 卡给应用**  
  在 Hyprland 配置中写入 N 卡的 Vulkan 环境变量（`VK_DRIVER_FILES` 等）。由于这些变量在桌面初始化后才加载，Hyprland 本身不受影响，但其子进程（游戏、浏览器等）会自动继承变量，从而透明地调用 N 卡进行渲染。

### 基本原理
桌面启动时 A 卡完成所有初始化，随后注入的 N 卡环境变量仅对 **后续启动的应用** 生效。钩子脚本通过监听虚拟机生命周期，自动处理驱动的绑定与解绑，实现“启动虚拟机则直通，关闭则归还”的无感切换，同时防止显卡占用冲突。

**注意事项：**
- 若使用 SDDM 等显示管理器，需确保其登录界面运行在 Wayland 模式下，否则会占用 N 卡。
- 副屏建议接在核显接口上。
- 归还 N 卡后出现的少量 nvidia 相关进程属于正常现象，不影响使用。
